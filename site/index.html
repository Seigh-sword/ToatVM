The VM that lives inside GitHub Actions

ToatVM boots a real operating system on a GitHub Actions runner and hands you a browser/terminal you can open anywhere. Driven entirely from your terminal with the toatvm CLI.

Install ToatVM View source

One command

npm i -g toatvm then toatvm -init.

Any OS

Ubuntu, Debian, Arch, Alpine, Fedora, Kali — in Docker.

Terminal + Desktop

A shell, or a full XFCE GUI over noVNC.

Templates

Save and reuse boot configs with toatvm -template.

Port forwards

Expose container ports via cloudflared tunnel.

Shareable

Hand a *.trycloudflare.com URL to a friend (password option).

Experimental. Runners are ephemeral and not private. Not for permanent or production use. Each session runs up to ~6h and restarts every cycle from a cached state.

Quick start

npm i -g toatvm
toatvm -new        # owner / repo / PAT (repo + workflow scopes)
toatvm -init       # pick OS, boot, get your live URL

How it works

toatvm calls the GitHub REST API with your PAT to dispatch a workflow. The runner launches a Docker container of your chosen OS, starts a shell (or desktop), and tunnels it to a public URL via cloudflared. The CLI reads the URL and credentials from the run's logs — no special token permission needed.

New in 0.3.0

  • Templates: save, list, apply, and delete session configs with toatvm -template
  • Environment variables: pass JSON env vars into the container with --env or toatvm -env
  • Pre-run / post-run scripts: run shell commands inside the VM on boot or after the cycle
  • Port forwards: expose host:container port mappings via the tunnel
  • Resource limits: set CPU, memory, and disk size for the container
  • Health checks: inspect run and job status with toatvm -health
  • Session history: view detailed history with toatvm -history
  • Import / export: migrate accounts between machines with toatvm -import and toatvm -export
  • Cleanup: cancel stale active runs with toatvm -cleanup
