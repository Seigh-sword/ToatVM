Choosing an OS

ToatVM runs your VM as a Docker container on the GitHub Actions runner, so you can pick almost any Linux distribution. Choose one with --os (or in the -init wizard).

ImageFlag valueNotes
Ubuntuubuntu:latestDefault. Familiar, lots of packages.
Debiandebian:latestStable, lightweight.
Arch Linuxarchlinux:latestRolling release, AUR.
Alpinealpine:latestTiny; uses apk/musl.
Fedorafedora:latestdnf, recent kernels.
Kalikalilinux/kali-rollingPen-test tooling preinstalled.

Example

toatvm -boot --mode t --os archlinux:latest --user neo --cycle 120

Windows?

The runner itself is Linux, so Windows is not offered as a container OS. On a Windows client you can still use ToatVM fully — the browser terminal works out of the box, and toatvm -open / toatvm -ssh integrate with Windows Terminal.

Your home directory is mounted from a cached volume, so files you create persist across the 1h cache-and-restart cycle.
